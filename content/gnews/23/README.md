## 解讀關於Zoom視頻聊天應用程序中所發現的各類安全問題以及最新事態發展
`2020-04-10 06:03 秘密翻译组` [轉載自GNews](https://gnews.org/zh-hant/168393/)

![](https://s3.amazonaws.com/gnews-media-offload/wp-content/uploads/2020/04/10042736/Picture-1-136.png)
美國科技新聞網站《CNET》4月9日，對Zoom迅速崛起以及已暴露出的一系列安全問題做了全面的時間列表。

在過去一個月中，由於中共冠狀病毒在全球的大爆發迫使數百萬人不得不呆在家裡。 Zoom也順理成章的成為首選的視頻會議服務。據報導，平台上的日常會議參與者從12月的1000萬激增至3月的2億。

伴隨著這種流行趨勢，Zoom的隱私風險也迅速引起人們的關注。從內置的跟踪功能到最近在“Zoombombing”中的騷動（指不請自來的與會者的闖入，並且散佈仇恨或色情的內容而破壞會議），Zoom對安全性的一些做法已引起了更多人的質疑。目前至少有三宗訴訟案是針對Zoom公司。

以下是對Zoom一連串在安全性方面所發生一切後的了解（以倒序的方式），以幫助不熟悉Zoom的安全性問題的讀者。

**4月9日**

- **美參議員被告知避免使用Zoom：**據《金融時報》週四報導，由於環繞視頻會議應用程序的安全問題，美國參議院告訴成員避免在冠狀病毒鎖定期間使用Zoom進行遠程工作。據報導，這不是官方的禁令，就像谷歌為其僱員發布的禁令一樣，但顯然要求參議員使用其它平台。


- **新加坡教師被禁止使用Zoom：**新加坡教育部表示，收到有關針對遠程學習學生的淫穢Zoombombing事件的報告後，教師已暫停使用Zoom。亞洲新聞頻道報導說，該部目前正在調查這些事件。


- **德國政府警告不能使用Zoom：**據德國報紙《 Handelsblatt》報導，德國外交部本週在內部通關中告訴員工，出於安全考慮，停止使用Zoom。衛生部在一份聲明中說，“由於與整個IT系統相關的風險，我們像其他部門和工業公司一樣，決定（聯邦外交部）不允許在用於商業目的的設備上使用Zoom。”


**4月8日**

- **第四起訴訟案件：**在周二向聯邦法院提起的訴訟中，Zoom股東邁克爾·德里厄（Michael Drieu）指控該公司“數據隱私和安全措施不足”，並錯誤地聲稱該服務是端到端加密的。德里厄還表示，有關安全問題的媒體報導和公司的公開承認，已導致Zoom的股價暴跌。


- **Google禁止使用Zoom：** Google在給員工的電子郵件中提到了安全漏洞，Google禁止在公司擁有的員工設備上使用Zoom，並警告說在本週內將停止該軟件在這些設備上運行。 Zoom是Google環聊會議應用程序的競爭對手。谷歌發言人在給BuzzFeed的電子郵件中說，在遠程工作時使用Zoom的員工需要查看其他地方，並且Zoom“不符合我們對員工使用的應用程序的安全標準。”


- **賞金獵人出現：**世界各地的黑客已開始轉向賞金狩獵，以尋找可出售給最高出價者的Zoom技術的潛在漏洞。一份母板（Motherboard）的報告中詳細說明了針對漏洞的弱點而提高賞金要價，有消息來源比哦啊是，目前黑客以5,000至30,000美元的價格出售這些漏洞。


**4月7日**

- **台灣禁止政府使用Zoom：**根據周二發布的一份聲明，出於安全方面的考慮，台灣政府機構被告知不能使用Zoom。台灣網絡安全部已授權使用Google和Microsoft等替代產品。


**4月6日**

- **一些學區禁止使用Zoom：**紐約一些學區開始禁止教師在冠狀病毒爆發期間使用Zoom進行遠程教學，理由是視頻會議應用程序存在著安全性和隱私問題。據Chalkbeat新聞網報導說，紐約教育局已敦促學校盡快改用其他替換軟件，例如Microsoft Teams應用程序。


- **暗網中找到Zoom的帳號：**網絡安全公司Sixgill透露，它在一個受歡迎的黑網論壇上發現了有人發布了352個被集體洩露的Zoom帳戶的鏈接。 Sixgill告訴雅虎財經，這些鏈接包括電子郵件地址，密碼，會議ID，主機密鑰和名稱以及Zoom帳戶的類型。大多數是個人的，但不是全部。 Sixgill告訴雅虎財經：“其中一個屬於美國主要的醫療保健服務提供者，另外七個屬於各個教育機構，另一個屬於小型企業。”


- **Zoom尋求在華盛頓擴大遊說活動：** Zoom對安全問題的回應主要集中在華盛頓特區。該公司告訴媒體Politico，它希望擴大在華盛頓的遊說活動，並聘請了布魯斯·梅爾曼（Bruce Mehlman），後者是總統喬治·W·布什（George W. Bush）任職前的技術政策商務助理部長。


- **敦促聯邦貿易委員會進行調查：**電子隱私信息中心在一封公開信中，敦促美聯邦貿易委員會調查Zoom，並發布視頻會議平台的隱私準則。美康涅狄格州民主黨參議員理查德·布盧門撒爾（Richard Blumenthal）最近為此事帶頭提出立法，儘管批評家說該立法可能削弱現代加密標準。他呼籲聯邦貿易委員會（FTC）對Zoom進行調查，稱其為“一個安全故障和侵犯隱私的典例”。


- **第三例訴訟：**在加利福尼亞，對Zoom發起了第三例訴訟，理由是研究人員提出了三個最重要的安全問題：與臉書Facebook數據共享，該公司承認的不完整的總端之間的加密，以及惡意行為者被允許進入用戶攝像頭的漏洞。


**4月5日**

- **用戶的撥叫會’錯誤地’通過中國白名單上的服務器：** Zoom在一份聲明中承認，一些視頻通話本不應該經過兩個在中共白名單上的服務器，但是被“錯誤地”傳送。它說，某些會議“被允許連接到中共的系統，而這些地方本不應該能夠被鏈接到”。


**4月4日**

- **又一個來自Zoom的道歉：** Zoom首席執行官埃里克·袁（Eric Yuan）在一個對《華爾街日報》長採訪中說：“我作為CEO，真的將事情搞砸了，我們需要贏回客戶的信任。這種事情本不應該發生。” 在談到公司被調查而聲譽受損時，袁講述了在中共病毒爆發初期，Zoom一味強求擴充而應對勞動人員的變化。


**4月3日**

- **Zoom視頻通話記錄，公開顯示在網絡上：** 《華盛頓郵報》的一項調查發現，成千上萬的Zoom視頻通話記錄沒有受到保護，在公開的網絡上可查看。報紙發現，大量未受保護的包括討論個人身份信息的通話，例如私人理療會議，遠程醫療培訓電話，討論私人公司財務報表的小型企業會議以及小學班級裡學生信息全部被曝光。


- **攻擊者計劃“ Zoomraids”：** 《CNET》和《紐約時報》的報導均顯示，匿名攻擊者正在利用包括Twitter和Instagram在內的社交媒體平台來組織“ Zoomraids”，這是協調大規模Zoombombings的術語，這些不速之客對會議參與者進行騷擾和謾罵。對Zoomraids攻擊時，在被報告的濫用行為中，包括使用種族主義，反猶太和色情圖片以及口頭騷擾。


- **Zoom再次道歉：** Zoom承認其自定義加密技術不合格。在一項Citizen Lab的報告發現該公司一直在私底下使用一種不太安全的AES-128密鑰，而不是先前聲稱將使用的AES-256加密方式。袁在直接回應中公開表示：“我們認識到我們可以在加密設計上做得更好。”


- **第二例訴訟：** Tycko和Zavareei LLP公司針對Zoom提起了集體訴訟，這是該公司的第二起訴訟，目的是與臉書Facebook共享用戶的個人信息。


- **國會要求提供信息：**美加州民主黨眾議員傑里·麥克納尼（Jerry McNerney）和眾議院能源與商業委員會的18名民主黨同事給Zoom CEO致信，對公司的對隱私的做法表示關注和疑問。這封信要求Zoom於4月10日前答复。


**4月2日**

- **自動搜索工具可以找到Zoom會議：**安全研究專家布萊恩·克雷布斯（Brian Krebs）報告說，安全研究人員透露，一個自動搜索工具可以在一小時內找到大約100個Zoom會議ID，並在一天的掃描中收集了近2,400個Zoom會議的信息。根據The Verge的報告，可發現的會議是不受密碼保護的，但該工具最多可以成功生成會議ID的時間達到14％。


- **在Zoombombing策劃更多攻擊計劃：**同時間，Motherboard稱發現8chan論壇用戶曾經計劃在一次反猶太Zoombombing行動中，強行控制費城一家猶太學校在Zoom上的會議啟動。


- **發現數據挖掘功能：**紐約時報報導說，Zoom上的數據挖掘功能，使一些參與者可以秘密訪問有關其他用戶在LinkedIn網上的個人資料數據。


**4月1日**

- **SpaceX禁止使用Zoom：**據《路透社》報導，埃隆·馬斯克（Elon Musk）的SpaceX火箭公司以“重大隱私和安全問題”為由，禁止員工使用Zoom。


- **發現更多安全漏洞：** Motherboard的報告再次揭示了Zoom的另一個破壞性安全漏洞，發現該應用程序曾通過沒有權限設計的一個公司目錄的功能，將用戶的電子郵件地址和照片洩露給陌生人。


- **來自Zoom的道歉：** Zoom首席執行官埃里克·袁（Eric Yuan）在博客中公開道歉，並發誓要提高安全性。其中包括啟用等候室和所有電話的密碼保護。袁還表示，該公司將凍結功能更新，以在未來90天內解決安全問題。


**3月30日**

- **The Intercept網調查：** Zoom沒有按照承諾使用終端之間加密。 The Intercept網的一項調查發現，Zoom通話數據被發送回該公司，而其營銷資料中有終端之間加密的承諾。


- **Zoom發言人告訴The Intercept：** “目前，不可能為Zoom視頻會議啟用E2E加密。”


- **發現更多漏洞：**發現與Windows相關的Zoom漏洞後，人們可以進行密碼盜竊，之前的NSA黑客又發現了另外兩個漏洞，其中一個漏洞可能允許惡意行為者控制Zoom用戶的麥克風或網絡攝像頭。另一個漏洞使Zoom可以在MacOS桌面上獲得用戶准入根權限，充其量是一個危險的准入級別。


- **提起第一例訴訟：** Zoom公司被集體訴訟，指控Zoom違反了加利福尼亞州的新數據保護法，未獲得用戶同意就將其Zoom上的數據傳輸到臉書Facebook。


- **紐約總檢察長的信已發送：**紐約總檢察長萊蒂蒂亞·詹姆斯（Letitia James）辦公室給Zoom致信，概述了隱私漏洞問題，並詢問該公司已採取哪些步驟（如果有的話）來確保用戶安全，考慮到其網絡流量的增加。


- **被報告在課堂上的Zoombombings：**報告發生在教室裡Zoombombing的案例，包括黑客闖入課堂會議，並在學生的屏幕上顯示納粹十字形的事件，導致FBI發出有關Zoom的安全漏洞的公共警告。該組織建議教育工作者使用密碼保護視頻通話，並使用軟件中當前可用的隱私功能，鎖定會議安全性。


**3月27日**

- **Zoom刪除了Facebook數據收集功能：**為了回應Motherboard的調查所提出的擔憂，Zoom從其iOS應用中刪除了Facebook數據收集功能，並在聲明中道歉。 Zoom告訴Motherboard，“ Facebook SDK收集的數據不包含任何個人用戶信息，而是包含有關用戶設備的數據，例如移動操作系統的類型和版本，設備時區，設備操作系統，設備型號和載體，屏幕尺寸，處理器核心和磁盤空間。”


**3月26日**

- **Motherboard調查：** Zoom的iOS應用程序將用戶數據發送到臉書Facebook。 Motherboard進行的一項調查顯示，Zoom的iOS應用程序正在通過與Facebook Graph API的交互將用戶分析數據發送到臉書Facebook，甚至對於沒有臉書Facebook帳戶的Zoom用戶也是如此。
